introduction

mockr is a CLI tool that spins up a fake REST API from a JSON config file. no backend, no setup — just define your endpoints and start building.

installation

mockr requires Node.js v14 or higher.

use without installing

terminal
npx mockr-cli start

install globally

terminal
npm install -g mockr-cli

once installed globally, run mockr-cli start from any folder.

quick start

up and running in 4 steps.
01
create a project folder
terminal
mkdir my-app
cd my-app
02
create api.json
api.json
{
  "GET /users": [{ "id": 1, "name": "Alice" }, { "id": 2, "name": "Bob" }],
  "POST /login": { "token": "abc123" }
}
03
start mockr
terminal
npx mockr-cli start
04
fetch from your frontend
javascript
fetch("http://localhost:3000/users")
  .then(res => res.json())
  .then(data => console.log(data))

api.json format

each key is a route, each value is the response.
api.json
{
  "GET /users": [...],
  "GET /users/1": {...},
  "POST /login": {...},
  "DELETE /users/1": { "success": true }
}

supported methods: GET POST PUT PATCH DELETE

the config file defaults to api.json in the current directory. use --config to point to a different file.

cli options

all flags are optional.
flagdefaultdescription
--config <file>api.jsonpath to your config file
--port <number>3000port to run the server on
--watchoffauto-reload when the config file changes
terminal
mockr-cli start --config mocks/api.json --port 8080 --watch

status codes

return any HTTP status code using _status.
api.json
{
  "GET /secret": { "_status": 403, "error": "Forbidden" },
  "GET /crash":  { "_status": 500, "error": "Internal Server Error" }
}
fields starting with _ are reserved for mockr and won't appear in the response body.

response delays

simulate slow networks using _delay (milliseconds).
api.json
{
  "GET /users": { "_delay": 1500, "data": [{ "id": 1, "name": "Alice" }] }
}

you can combine _delay and _status:

api.json
{
  "GET /slow-error": { "_delay": 2000, "_status": 503, "error": "Service Unavailable" }
}

watch mode

auto-reload on config changes.
terminal
mockr-cli start --watch

when you save your config file, mockr reloads automatically — no restart needed.

request logging

every request is logged automatically.
terminal output
  10:32:01  200  GET /users
  10:32:04  404  GET /unknown
  10:32:09  200  GET /users  (1500ms delay)
  10:32:11  403  GET /secret

examples

copy-paste ready configs.

basic REST API

api.json
{
  "GET /posts": [
    { "id": 1, "title": "Hello World", "author": "Alice" },
    { "id": 2, "title": "Second Post", "author": "Bob" }
  ],
  "GET /posts/1": { "id": 1, "title": "Hello World", "body": "...", "author": "Alice" },
  "POST /posts": { "id": 3, "message": "created" },
  "DELETE /posts/1": { "success": true }
}

auth flow

api.json
{
  "POST /login": { "token": "eyJhbGciOiJIUzI1NiJ9", "expires": 3600 },
  "GET /me": { "id": 1, "name": "Alice", "email": "alice@example.com" },
  "POST /logout": { "success": true }
}

error handling

api.json
{
  "GET /admin": { "_status": 403, "error": "Forbidden" },
  "GET /timeout": { "_delay": 5000, "_status": 408, "error": "Request Timeout" },
  "GET /server-error": { "_status": 500, "error": "Something went wrong" }
}
